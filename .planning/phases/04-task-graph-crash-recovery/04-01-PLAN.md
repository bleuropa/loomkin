    - Test 2: After a crashed process is replaced by a new process with the same registry key, AgentWatcher detects recovery and publishes Recovered signal
    - Test 3: When recovery check exceeds max retries (5 checks), AgentWatcher publishes PermanentlyFailed signal
    - Test 4: Crash count increments across multiple crashes of the same agent

  <action>
Create `lib/loomkin/teams/agent_watcher.ex`:
- `use GenServer` with state: `%{agents: %{}, team_id: nil}`
- `start_link(opts)` -- takes `team_id` option, registers via `{:via, Registry, {Loomkin.Teams.WatcherRegistry, team_id}}` or simple name
- `watch(watcher, pid, team_id, agent_name)` -- `GenServer.cast` to monitor the pid via `Process.monitor/1`, store `{pid, team_id, agent_name, crash_count}` keyed by monitor ref
- `handle_info({:DOWN, ref, :process, pid, reason})` -- on process death: pop from agents map, increment crash_count, publish Crashed signal via `Jido.Signal.Bus.publish/2`, schedule `{:check_recovery, team_id, agent_name, crash_count}` via `Process.send_after(self(), msg, 500)`
- `handle_info({:check_recovery, team_id, name, crash_count})` -- look up agent in `Loomkin.Teams.AgentRegistry` via `Registry.lookup/2`. If found: publish Recovered signal, re-monitor new pid, store with updated crash_count. If not found and crash_count < 5: retry with `Process.send_after(self(), msg, 500)`. If crash_count >= 5: publish PermanentlyFailed signal
- Signal publishing: use the bus pattern from the codebase -- `Jido.Signal.Bus.publish(bus_name, signal)` where bus_name matches the team's bus. Check how `set_status_and_broadcast` in Agent publishes signals and follow the same bus resolution pattern.

Update `lib/loomkin/teams/agent.ex` `handle_info({:DOWN, ...})` (line 815):
- Change `set_status_and_broadcast(state, :idle)` to `set_status_and_broadcast(state, :error)` when the reason is not `:normal` or `:shutdown`. This makes the Agent GenServer itself report an error status on loop crashes (distinct from the AgentWatcher process-level crash detection). Keep `:idle` for normal/shutdown exits.

Note: AgentWatcher is for detecting GenServer-level process crashes (when the Agent GenServer itself dies and DynamicSupervisor restarts it). The Agent's internal :DOWN handler is for loop_task crashes (the Task.async that runs the agent loop). Both are needed.

Run `mix format` before committing.
