```elixir
def list_with_deps(team_id) do
  tasks = list_all(team_id)
  task_ids = Enum.map(tasks, & &1.id)

  deps =
    if task_ids == [] do
      []
    else
      Repo.all(
        from d in TeamTaskDep,
          where: d.task_id in ^task_ids or d.depends_on_id in ^task_ids
      )
    end

  {tasks, deps}
end
```

**Module structure:**
- `use LoomkinWeb, :live_component`
- Constants: `@layer_gap 120`, `@node_gap 180`, `@node_width 140`, `@node_height 60`
- Task status colors map: `@status_colors %{pending: {"#9ca3af", "#374151"}, assigned: {"#60a5fa", "#1e3a5f"}, in_progress: {"#fbbf24", "#78350f"}, completed: {"#4ade80", "#166534"}, failed: {"#f87171", "#7f1d1d"}, blocked: {"#fb923c", "#7c2d12"}}`

**mount/1:** Initialize with empty nodes, edges, positioned, selected_node, svg_width, svg_height, critical_path_edges (MapSet)

**update/2:** Check `refresh_ref` changes (same pattern as DecisionGraphComponent). When changed, call `do_load_graph/3` which:
1. Calls `Tasks.list_with_deps(team_id)` to get `{tasks, deps}`
2. Calls `layout_tasks(tasks, deps)` for positioning
3. Calls `compute_critical_path(tasks, deps)` for critical path edge set
4. Builds edge list from deps with source/target positions
5. Calls `compute_svg_dimensions(positioned)`

**layout_tasks/2:** Topological layering based on blocking dependency depth:
1. `compute_depths(tasks, deps)` -- BFS from root tasks (no blocking dependencies) assigning depth layers. For each blocking dep, the dependent task is one layer deeper than its dependency
2. Group tasks by depth layer
3. Position: x = 40 + index_in_layer * @node_gap, y = 40 + layer * @layer_gap

**compute_critical_path/2:** Find the longest chain of blocking deps among incomplete tasks:
1. Filter to `:blocks` deps only, among incomplete tasks
2. Build adjacency map (dependency -> list of dependent tasks)
3. Find roots (tasks with no incoming blocking deps among incomplete set)
4. DFS from each root computing longest path
5. Return MapSet of `{from_id, to_id}` edge tuples on the critical path

**render/1:** SVG with:
- `<svg>` element with computed width/height, dark background
- Edge rendering: For each dep edge, draw cubic bezier path (same curve math as DecisionGraphComponent). Solid `stroke-dasharray: none` for `:blocks`, `stroke-dasharray: 6,4` for `:informs`. If edge is in critical_path_edges MapSet, use thicker stroke (3px vs 1.5px) and brighter color (#f59e0b)

- Arrowhead marker defs (same as DecisionGraphComponent)

**handle_event("select_node", ...):** Set selected_node assign, show detail panel
**handle_event("close_detail", ...):** Clear selected_node

**Detail panel (below SVG):** Shows selected task's title, description, status, assigned agent, dependencies list, and result (if completed/failed). Same layout pattern as DecisionGraphComponent's node_detail.

Follow DecisionGraphComponent's SVG rendering patterns closely for consistency. Reuse the same bezier curve calculation for edges. Do NOT implement: task filtering, task actions (reassign/cancel), progress bars -- these are deferred.
